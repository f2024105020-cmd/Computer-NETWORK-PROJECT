# 🌐 Computer Network Project – Web Traffic Monitoring & Security

> **Secure communication matters** – this project demonstrates why we must replace Telnet with SSH and HTTP with HTTPS.

---

## 📌 Overview
- **Objective**: Monitor web traffic and highlight security risks of unencrypted protocols.
- **Tools Used**: Cisco Packet Tracer, Wireshark (or your actual tools).
- **Key Concepts**: Encryption, SSH vs Telnet, HTTPS vs HTTP, packet sniffing.

---

## 🚀 How to Run the Simulation
1. Open `web monitoring 4 final......pkt` in Cisco Packet Tracer.
2. Start the devices and generate HTTP/HTTPS traffic.
3. Observe how HTTP traffic is transmitted in plaintext while HTTPS is encrypted.

---

## 🔒 Security Takeaways
- **Telnet** sends credentials in plaintext → use **SSH**.
- **HTTP** exposes data → use **HTTPS**.
- Network monitoring helps detect vulnerabilities early.
